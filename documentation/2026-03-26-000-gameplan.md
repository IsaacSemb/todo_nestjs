# Gameplan (Monorepo + NestJS TODO) - 10-hour learning track

## Phase 0 - Monorepo foundation (Hour 1)
- Use one repo with npm workspaces and clear boundaries:
  - `apps/client` (React + Vite + strict TS)
  - `apps/server` (NestJS + strict TS)
  - `packages/*` for shared config/types/contracts
- Establish strict TypeScript baseline across all workspaces.
- Add shared lint/format standards.
- Add root scripts for dev/build/lint/typecheck/test.

Deliverable: both apps run, strict TS is active, root scripts work.

## Phase 1 - Nest core concepts (Hour 2)
- Implement Nest fundamentals:
  - Modules, Controllers, Providers, Dependency Injection
  - Global validation with DTOs
- Create:
  - `HealthModule` (`GET /health`)
  - `TodosModule` with in-memory storage first
- Learn exception flow and status code mapping.

Deliverable: validated in-memory todo endpoints with clean Nest structure.

## Phase 2 - Prisma integration (Hour 3)
- Add Prisma to `apps/server`.
- Define first schema:
  - `User`
  - `Todo` (owner relation, status, timestamps)
- Create Prisma module/service and refactor todos to DB-backed flow.
- Add migration and basic seed strategy.

Deliverable: todo data persisted in a real database.

## Phase 3 - REST discipline (Hour 4)
- Build complete CRUD quality:
  - pagination
  - filtering
  - sorting
- Keep response/error shape consistent.
- Add Swagger/OpenAPI docs.

Deliverable: production-style REST API surface for todos.

## Phase 4 - Authentication (Hour 5)
- Start with JWT auth:
  - register/login/refresh/logout
  - `GET /me`
- Use guards/strategies properly.
- Decide token storage and refresh flow for SPA.

Deliverable: authenticated user flow end-to-end.

## Phase 5 - Authorization and security (Hour 6)
- Enforce ownership rules: users only access their own todos.
- Add basic RBAC concepts (minimal but real).
- Add security baseline:
  - CORS policy
  - security headers
  - rate limiting (if time allows)

Deliverable: secure access model with clear authorization checks.

## Phase 6 - Frontend integration (Hour 7)
- Build typed API client in React.
- Add auth-aware app state and protected routes.
- Connect UI to real backend contracts.

Deliverable: full React <-> Nest flow for todos and auth.

## Phase 7 - Testing strategy (Hour 8)
- Backend unit tests (services/guards).
- Backend e2e tests (auth + todo happy/edge paths).
- Add repeatable test data setup/reset.
- Optional frontend integration tests for critical flows.

Deliverable: regression safety net for core behavior.

## Phase 8 - Production patterns (Hour 9)
- Validate env config and fail fast on invalid startup config.
- Add global exception handling and normalized error contracts.
- Add structured logging and request-level observability.
- Add caching selectively where useful.

Deliverable: app behaves like a real service, not a demo script.

## Phase 9 - Intentional over-engineering (Hour 10)
- Add one advanced subsystem:
  - audit logs
  - soft deletes
  - optimistic concurrency
  - background jobs/events
- Review architecture tradeoffs and simplification paths.

Deliverable: practical understanding of design tradeoffs in Nest systems.

## Rules for every phase
- Lint passes.
- Typecheck passes.
- Minimum tests for the phase pass.
- Brief write-up: what changed, why it matters, and tradeoffs.
